
EMPIRICAL TIER (the bar a method must clear to count as empirical)
  three structural requirements. all three. anything missing -> not empirical.

  1. RUNNABLE.  the method requires --cmd. clms executes it at verify
                time and records actual exit_code + stdout_hash. drift is
                mechanically detectable on rerun.
  2. STRUCTURAL PASS/FAIL.  clms enforces a specific shape on the
                evidence row that can refuse the verify outright:
                  prop/integration/replay  exit_code == 0
                  stat-test                p in [0, 1] AND sample_size >= 2
                  benchmark                metric_value vs threshold in
                                           the direction declared per metric
                  estimate                 ci_lower <= point_value <= ci_upper
                                           AND 0 < confidence_level < 1
  3. REAL DATA.  --data-source must be real | live; simulated/synthetic
                is refused at parse time. methods that don't take
                --data-source (prop/integration/replay) carry the same
                discipline structurally (real generator, real external
                system at --target, real-world capture at --dataset).

  the six methods below all clear this bar. observed / documented /
  derived do not (no --cmd execution, no structural shape on the artifact
  beyond well-formedness, no real-data requirement).

EVIDENCE METHODS (each has REQUIRED fields, no soft warnings, exit 1 if missing)
  prop-test         --ref --cmd                       [--exit-code predictive]
  integration-test  --ref --cmd --target <url-or-host> [--exit-code predictive]
  replay-test       --ref --cmd --dataset <path>      [--exit-code predictive]
  stat-test         --ref --cmd                       [--test-type --p-value
                    --sample-size --data-source as optional echo-checks]
                    --ref must be a LOCAL JSON artifact produced by --cmd.
                    artifact fields required: test_type, p_value, sample_size,
                    data_source. if the matching cli flags are supplied, they
                    must match the artifact exactly.
                    --test-type: closed enum, one of:
                      chi-squared | chi-squared-goodness-of-fit
                      t-test-paired | t-test-unpaired | t-test-one-sample | welch-t
                      anova | kolmogorov-smirnov | mann-whitney-u | wilcoxon-signed-rank
                      shapiro-wilk | anderson-darling | fisher | permutation | likelihood-ratio
  benchmark         --ref --cmd --threshold          [--metric --metric-value
                    --sample-size --data-source as optional echo-checks]
                    --ref must be a LOCAL JSON artifact produced by --cmd.
                    artifact fields required: metric, metric_value, sample_size,
                    data_source.
                    --metric: closed enum, direction fixed per variant.
                      higher-better: auc-roc | auc-pr | f1 | precision | recall | accuracy |
                                     balanced-accuracy | mcc | kappa-cohen | r2
                      lower-better:  log-loss | brier | rmse | mae | mape
                    gate: artifact.metric_value >= threshold (higher-better)
                          or <= threshold (lower-better)
                    cmd execution: --cmd is run at verify time. exit_code must be 0.
                    stdout_hash captured for rerun drift detection.
  estimate          --ref --cmd                      [--estimator --point-value
                    --ci-lower --ci-upper --confidence-level --sample-size
                    --data-source as optional echo-checks]
                    --ref must be a LOCAL JSON artifact produced by --cmd.
                    artifact fields required: estimator, point_value, ci_lower,
                    ci_upper, confidence_level, sample_size, data_source.
                    --estimator: closed enum, one of:
                      mean | median | geometric-mean | std-dev | std-error | variance
                      skewness | kurtosis | cohens-d | odds-ratio | risk-ratio
                      correlation | spearman-rho
                    gate: artifact.ci_lower <= artifact.point_value <= artifact.ci_upper
                          AND 0 < artifact.confidence_level < 1.
                    cmd execution: --cmd is run at verify time. exit_code must be 0.
                    stdout_hash captured for rerun drift detection.
  observed          --ref                              (file | URL | sha256:HEX)
  documented        --ref --quote "<exact text from doc>"
  derived           --from <id> --from <id>            (min 2, each must exist + be verified)

CMD EXECUTION (all empirical methods)
  --cmd is EXECUTED at verify time. clms captures the actual exit_code and
  hashes stdout.

  - prop-test / integration-test / replay-test:
      --exit-code is optional and reinterpreted as the agent's *predicted*
      exit. mismatch with actual is a hard error.
  - stat-test / benchmark / estimate:
      --cmd is REQUIRED and must exit 0. clms then parses the LOCAL JSON
      artifact at --ref and treats that artifact as the source of truth.
      matching cli flags, if present, are only consistency checks.

REF / TARGET SHAPE
  observed --ref must be one of: (a) an existing local file path, (b) a URL
  with a recognized scheme (http/https/file/ftp/ssh/git/ws/wss), or (c) a
  content-address like sha256:HEX / blake3:HEX. bare strings are refused.

  integration-test --target must be a real external system. loopback (127.x,
  ::1, localhost), RFC1918 (10.x, 192.168.x, 172.16-31.x), link-local, and
  unspecified addresses are refused. pass --allow-local if you intentionally
  want to record a local smoketest as integration evidence.

REFUSED METHODS (parse-time error)
  unit-test  confirmatory only. you picked the input AND the expected output;
             the test cannot disagree with you. use prop-test for randomized
             input exploration, integration-test for real systems, replay-test
             for real captured data, or observed for a single artifact.
  code-test  removed in schema 1.1. ambiguous about falsification surface.
             pick one of: prop-test | integration-test | replay-test.
  sim-test   synthetic data is circular w/ the assumptions you're testing.
             use stat-test --data-source=real, or file as derived citing the
             simulator's underlying claims.

EXAMPLES
  # property-based test (randomized input generator)
  # --cmd is executed; --exit-code omitted, actual captured silently.
  clms verify 1 --method prop-test --ref ./tests/sort_props.rs \
    --cmd "cargo test --release sort_props -- --nocapture"

  # same, but with predicted exit asserted (mismatch → hard error)
  clms verify 1 --method prop-test --ref ./tests/sort_props.rs --exit-code 0 \
    --cmd "cargo test --release sort_props -- --nocapture"

  # integration test against a real external system
  clms verify 2 --method integration-test --ref ./tests/poly_api.py \
    --target https://clob.polymarket.com \
    --cmd "python3 ./tests/poly_api.py"

  # local CI smoketest (lower confidence; explicit --allow-local)
  clms verify 2 --method integration-test --ref ./tests/health.sh \
    --target http://localhost:8080 --allow-local \
    --cmd "./tests/health.sh"

  # replay test against frozen real-world capture
  clms verify 3 --method replay-test --ref ./strategies/mm_v3.py \
    --dataset ./data/binance_btc_l2_2024-03.parquet \
    --cmd "python3 ./strategies/mm_v3.py --replay ./data/binance_btc_l2_2024-03.parquet"

  # statistical test on real samples (artifact is source of truth)
  clms verify 4 --method stat-test --ref ./lag_test_artifact.json \
    --cmd "python3 ./test.py --out ./lag_test_artifact.json"

  # benchmark: classifier evaluation against threshold
  clms verify 5 --method benchmark --ref ./eval_output.json \
    --threshold 0.80 \
    --cmd "python3 ./eval.py --holdout ./data/test.parquet --out ./eval_output.json"

  # estimate: mean with 95% CI on real returns
  clms verify 6 --method estimate --ref ./estimator_out.json \
    --cmd "python3 ./est.py --series ./data/eth_1h_returns.parquet --out ./estimator_out.json"

  # documented source
  clms verify 7 --method documented --ref https://docs.poly.com/api/limits \
    --quote "default rate limit is 100 requests per minute"

FALSIFICATION SURFACES
  prop-test         randomized input generator finds counterexamples
  integration-test  the real external system at --target can disagree
  replay-test       frozen real-world data at --dataset can disagree
  stat-test         real/live samples can fail to reject the null
  benchmark         measured metric on held-out real data can miss the threshold
  estimate          replicated computation can yield a CI disjoint from the claimed one
  observed          the artifact can be missing or contradictory
  documented        the quote can be missing, edited, or contradicted
  derived           upstream claims can be refuted (cascade)

MIN_TIER GATE
  if the claim was created with `clms add --min-tier <tier>`, verify refuses
  to mark it Verified using evidence below that floor. tier ladder (strongest
  to weakest): empirical > observed > documented > derived. example: a claim
  stamped `min_tier=empirical` cannot be Verified by `--method observed`,
  `--method documented`, or `--method derived` — only by prop-test,
  integration-test, replay-test, stat-test, benchmark, or estimate. the
  error lists the eligible methods at the declared floor, filtered from the
  METHODS table at runtime so it stays correct as the empirical bucket
  grows. claims without min_tier behave exactly as before (no gate, all
  methods accepted).

DRIFT (three checks, all gated by --acknowledge-drift)
  - ref drift:      same --ref string, different content hash. the file
                    behind the ref changed since prior evidence.
  - dataset drift:  same --dataset path, different content hash. the bytes
                    behind the path changed (parquet/csv swap).
  - rename drift:   same content hash, different --ref string. the agent
                    copied/renamed/symlinked an existing file under a new
                    path; the bytes are identical but the dedup keyed on
                    the ref string would not catch it.
  any drift refuses with exit 1 and shows the prior hash + recorded_at. add
  --acknowledge-drift if the iteration is intentional. if the new result
  contradicts the prior conclusion, you should refute the claim and write a
  new one instead of acknowledging drift.

ENV VARS (provenance + repair)
  CLAIMS_BACKFILL=1   required to use --git-sha or --created-at overrides on
                      `clms add`. claims born under this flag are stamped
                      `backfilled: true` and rendered with [backfilled] in
                      human output.
  CLAIMS_REPAIR=1     disables claim-integrity verification in read_claim for
                      forensic recovery (show / diff-evidence / timeline /
                      context). ALL mutating commands refuse under repair
                      mode, including add / verify / rerun / refute / reindex /
                      archaeology commit+purge.
  CLAIMS_INTEGRITY_KEY_FILE
                      path to keyed integrity material. defaults to
                      ~/.clms/integrity.key and is auto-created on first write.
  CLAIMS_INTEGRITY_KEY_HEX
                      32-byte (64 hex chars) integrity key supplied directly
                      via env. overrides CLAIMS_INTEGRITY_KEY_FILE.

CONFIDENCE
  auto-derived from method. cannot be set manually. add stronger evidence
  later (e.g. stat-test on top of documented) to bump confidence tier.
